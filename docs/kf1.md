# `kf1.py` Explanation

対象ファイル: [section_kalman_filter/kf1.py](../section_kalman_filter/kf1.py)

## Overview

2次元の地図の中でロボットを動かしながら、「自分はいまどこにいるか」を推定するシミュレーションです。

ロボットは地図上のランドマークを観測し、その観測結果を使って自己位置を更新します。  
この推定に使っているのが `Mcl` です。`Mcl` は Monte Carlo Localization の略で、たくさんの位置候補を持ちながら推定を行います。

## First Mental Model

1. 世界がある
2. 世界の中でロボットが動く
3. ロボットは目印を見ながら、自分の位置を推定する

もう少し具体的に言うと、ロボットには次の2つがあります。

- 本当の位置
- 推定している位置

本当の位置はロボットが実際に動いて変わります。  
推定している位置は、センサ観測をもとに `Mcl` が更新します。

## Main Characters

- `World`
  シミュレーション全体を管理します。時間を進め、地図やロボットを描画します。
- `Map`
  ランドマークを持つ地図です。
- `Landmark`
  ロボットが観測する目印です。
- `Robot`
  実際に動くロボット本体です。
- `Camera`
  ランドマークまでの距離と方向を観測するセンサです。
- `Mcl`
  自己位置推定器です。たくさんのパーティクルを使って「自分がいそうな場所」を表します。
- `EstimationAgent`
  ロボットに速度指令を出しつつ、`Mcl` を更新する役です。

## Overall Flow

`trial()` では大きく次の流れで処理が進みます。

1. シミュレーション世界を作る
2. 地図を作り、ランドマークを置く
3. ロボットの初期位置を決める
4. MCL を初期化する
5. ロボットにセンサと推定器を載せる
6. 時間を少しずつ進めながら、
   ロボットを動かし、目印を観測し、推定結果を更新する
7. それを画面に描画する

コードではこの部分が中心です。

```python
initial_pose = np.array([0, 0, 0]).T
estimator = Mcl(m, initial_pose, 100)
a = EstimationAgent(time_interval, 0.2, 10.0/180*math.pi, estimator)
r = Robot(initial_pose, sensor=Camera(m), agent=a, color="red")
world.append(r)
world.draw()
```

それぞれの意味は次の通りです。

- `initial_pose`
  ロボットの初期姿勢です。`[x, y, theta]` で表します。
- `Mcl(m, initial_pose, 100)`
  地図 `m` を使い、100個のパーティクルで自己位置推定を行います。
- `EstimationAgent(...)`
  ロボットを前進・回転させながら、MCL の更新も行います。
- `Robot(...)`
  本体、センサ、制御役をまとめたロボットを作ります。
- `world.draw()`
  シミュレーションを開始し、各時刻の処理を繰り返します。

## What Happens In One Step

このシミュレーションでいちばん大事なのは、「1回の時間更新で何が起きるか」です。  
1ステップでは次のことが起きます。

1. ロボットが現在位置からランドマークを見る
2. `Camera` がランドマークまでの距離と方向を観測する
3. `EstimationAgent` が MCL に「前回このくらい動いた」と伝える
4. MCL は全パーティクルを運動モデルに従って動かす
5. MCL は各パーティクルの位置から見たとき、ランドマークがどう見えるはずかを計算する
6. その予測と実際の観測を比べて、合っているパーティクルの重みを上げる
7. 重みの高いパーティクルを残しやすいようにリサンプリングする
8. もっともらしい位置を現在の推定結果として扱う
9. ロボット本体も速度指令に従って少し移動する

ここで重要なのは、MCL は「観測したから答えが1つ出る」のではなく、「観測に合う候補を残していく」方式だという点です。

## How The Robot Moves

`kf1.py` では、ロボットは推定結果を使って進路を変えているわけではありません。  
移動のしかたは最初から固定されています。

```python
a = EstimationAgent(time_interval, 0.2, 10.0/180*math.pi, estimator)
```

この2つの値がそのまま移動指令です。

- `0.2`
  前進速度 `nu`
- `10.0/180*math.pi`
  角速度 `omega`。これは 10 度/秒をラジアンに直した値です。

つまり、ロボットは毎ステップ

- 少し前進し
- 少し回転する

という固定的な動きを続けます。

MCL の推定結果は「いま自分がどこにいると考えているか」を表示するために使われていますが、その推定結果を見て右へ曲がる、止まる、目標へ向かう、といった制御はまだ行っていません。

したがって `kf1.py` は、

- ロボットを固定の速度で動かしながら
- MCL がその位置をどのように推定するかを見る

ためのシミュレーションです。

## What MCL Is Doing

`Mcl` の内部では、ロボット位置の候補をたくさん持っています。  
この候補1つ1つが `Particle` です。

各パーティクルは「私はここにいるかもしれない」という仮説です。  
MCL は毎ステップ、この仮説たちを次のように更新します。

1. ロボットが動いたので、各パーティクルも少し動かす
2. そのパーティクル位置からランドマークがどう見えるかを計算する
3. 実際のセンサ観測と比べる
4. 合っているパーティクルの重みを大きくし、合っていないものを小さくする
5. 重みの大きいものを多く残す

これを何度も繰り返すと、パーティクルが本当の位置の近くに集まってきます。

## Example Of One Observation

たとえばロボットがランドマークを見て、

- 距離 3.0
- 少し右方向

と観測したとします。

すると MCL は、各パーティクルについて次を考えます。

- もし自分がこの位置にいたら、そのランドマークはどの距離・方向に見えるはずか

その予測が実際の観測に近いパーティクルは、「その位置仮説はもっともらしい」と判断されます。  
逆に、まったく違う見え方になるパーティクルは「その位置仮説は怪しい」と判断されます。

つまり、ランドマーク観測は「正しい位置を直接返す」のではなく、「正しそうな位置候補を選別する材料」です。

## How The Files Are Connected

`kf1.py` は次のコードで `scripts` ディレクトリを import path に追加しています。

```python
import sys
from pathlib import Path

sys.path.append(str(Path(__file__).resolve().parent.parent / "scripts"))
from mcl import *
```

その後の依存関係は次の通りです。

- `kf1.py` が `mcl.py` を読む
- `mcl.py` が `robot.py` を読む
- `robot.py` が `ideal_robot.py` を読む

そのため、`kf1.py` では `World`, `Map`, `Landmark`, `Robot`, `Camera`, `Mcl`, `EstimationAgent` などをまとめて使えます。
